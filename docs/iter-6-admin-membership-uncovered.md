## 无法由单元测试完整覆盖的场景

### 1. 管理端真实会话与路由可见性

原因：依赖浏览器中的真实登录会话、Next.js 客户端路由和管理员身份刷新。

手动步骤：

1. 使用普通用户登录并访问 `/admin/membership`，确认管理内容不可见。
2. 使用全局管理员登录并访问 `/admin/users`，确认侧边栏出现 Membership。
3. 打开 `/admin/membership`，确认五档套餐、价格、核心配额和权益状态完整显示。

### 2. 管理端响应式和横向矩阵布局

原因：表格宽度、横向滚动和小屏标签布局属于真实浏览器视觉行为。

手动步骤：

1. 在桌面宽度检查权益矩阵列对齐和页面可读性。
2. 在 390 x 844 视口检查页面不产生整页横向溢出。
3. 确认权益矩阵只在自身容器内横向滚动。
4. 确认用户详情四个标签在小屏中不重叠。

### 3. 外部支付协议一致性

原因：管理员本地权益变更刻意不调用 PayPal，无法在单元测试中验证真实外部协议状态。

手动步骤：

1. 对具有 PayPal 订阅的租户打开 Membership 标签。
2. 确认页面显示支付渠道、周期结束时间和到期取消状态。
3. 调整本地套餐后确认 PayPal 外部协议未被修改。
4. 由运营人员按账单系统记录核对本地权益变更的业务依据。
