# 迭代 17: OAuth 账户绑定安全修复 - 未覆盖测试说明

## 未覆盖场景

- 现有本地账号与 OAuth 身份的显式绑定界面尚未实现；本次修复只阻止 OAuth 首次登录按同名用户名自动合并。
- 真实 Google、GitHub 邮箱所有权变更和邮箱回收属于第三方身份提供商生命周期，不能通过单元测试模拟。
- `createRegisteredUser` 位于既有的大型用户查询模块中；本次定向覆盖验证了 OAuth 分支和账户创建参数，但单文件历史代码无法以本次迭代达到整体 95% 语句覆盖率。

## 手动测试步骤

1. 创建一个本地密码账号，其用户名使用邮箱格式。
2. 用同一邮箱的首次 Google 或 GitHub OAuth 登录，确认系统创建独立 OAuth 账号，不返回或访问原本地账号的数据。
3. 再次使用相同的 OAuth provider account 登录，确认稳定返回新 OAuth 账号。
4. 确认登录、OAuth callback 和 exchange 日志中不输出访问令牌、授权码或密码。

## 建议补充的测试用例

- 添加已登录设置页的显式 OAuth 绑定功能后，覆盖密码再认证、冲突处理、解绑和审计事件。
- 在预生产对 Google 和 GitHub 分别完成一次真实授权与重复登录回归。
