# Iteration 8: Activation codes

## 已覆盖的真实业务链路

- `route.integration.test.ts` 使用真实 `parseRequest`、管理 API Route、Prisma 和 PostgreSQL，覆盖创建、列表、详情、更新、软删除及测试数据清理；仅模拟登录认证边界。
- 创建请求包含浏览器表单实际发送的 `name: null`、`note: null`、ISO 时间和空失效时间语义，避免单元测试 mock 掩盖前后端契约不一致。
- 新创建的完整激活码使用 AES-256-GCM 加密保存，兑换仍按不可逆哈希匹配；管理员列表和详情 API 解密后可重复显示，密文和哈希不会进入响应。
- 迁移前的历史激活码没有可恢复密文，管理端仅显示前缀并明确标记完整码不可用。
- 集成测试默认跳过，防止普通单元测试误连开发数据库。显式运行命令：

```bash
RUN_DATABASE_INTEGRATION_TESTS=1 pnpm exec dotenv -- vitest run \
  src/app/api/admin/activation-codes/route.integration.test.ts
```

## 未覆盖场景

- React Aria 的创建、编辑、删除和详情弹窗在真实浏览器中的焦点管理、键盘操作和移动端布局。
- 创建成功后激活码仅显示一次，`allowCopy` 调用浏览器剪贴板权限失败或被拒绝时的交互。
- PostgreSQL 真实迁移后的并发兑换，包括多个用户同时消耗最后一个名额，以及同一用户并发提交同一个激活码。
- 管理员降低使用上限与用户兑换同时发生，以及同一用户用不同激活码并发延长会员时长的事务交错。
- 激活码会员到期后由真实租户请求触发回落和数据保留策略更新。
- 多租户用户在切换当前工作区后兑换激活码，确认权益只发放到请求中的工作区且普通成员无法绕过账单管理权限。
- PayPal 返回有效订阅但缺少 `currentPeriodEnd` 时的真实账单状态同步。
- 52 个 locale 的真实页面排版，尤其是长译文在窄屏表格、表单和升级页按钮中的换行。

## 手动测试步骤

1. 执行 `pnpm prisma migrate deploy`，启动应用并以管理员登录 `/admin/activation-codes`。
2. 创建一个 `pro`、7 天有效、启用时间为当前、最大使用次数为 2 的激活码，复制创建成功后显示的完整码。
3. 在两个不同用户的 `/membership/upgrade` 页面使用同一激活码，确认两次兑换成功；使用第三个用户确认收到次数上限错误，并确认权益发放到页面当前工作区。
4. 使用同一用户再次兑换，确认收到重复兑换错误；设置未来启用时间和过去失效时间分别验证对应错误。
5. 使用只有普通成员权限的账号尝试兑换，确认返回 403；使用低于当前有效会员等级的激活码，确认不会发生降级。
6. 在当前激活码会员仍有剩余时长时兑换相同或更高等级激活码，确认新时长从原到期日累加。
7. 编辑激活码为停用并删除，确认列表状态变化、详情兑换记录保留且删除后不能继续兑换。
8. 在英文、简体中文和至少一个长文本 locale 下重复步骤 1-7，检查表单、表格、按钮和弹窗无重叠。

## 建议补充的测试用例

- 使用 Playwright 覆盖管理员表单完整提交、一次性展示、剪贴板复制和键盘关闭弹窗。
- 使用 PostgreSQL 集成测试验证 `redemptionCount` 原子递增及 `activation_code_redemption` 唯一约束在并发请求下的行为。
- 增加管理员降低上限与兑换并发执行的集成测试，验证 `redemptionCount <= maxRedemptions` 始终成立。
- 增加不同激活码并发兑换的集成测试，验证会员剩余时长不会因事务交错丢失。
- 增加到期租户的 API 集成测试，验证 plan、subscription 和 retention cutoff 同步回落。
- 为每个支持的 locale 增加窄屏截图回归，锁定激活码按钮和管理表格的换行行为。
