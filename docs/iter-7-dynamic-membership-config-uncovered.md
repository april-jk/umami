# 迭代 7：动态会员配置未覆盖说明

## 已完成的自动化覆盖

- 全局会员配置的校验、默认值回退、数据库持久化、Redis/内存缓存和版本冲突。
- 仅全局管理员可读取和保存完整配置，公共接口不暴露更新人。
- 管理端价格、套餐可用性、无限量数值和布尔权益的编辑、保存、放弃与错误状态。
- 升级页动态价格、年付月均价、配额、权益数量、套餐隐藏和推荐套餐计算。
- 52 个语言的 12 项动态权益标签结构与非空翻译校验。
- 网站、成员、事件、Goal 和数据保留对动态配置的真实后端拦截。
- 所有本次新增或修改的核心源文件语句覆盖率不低于 95%。

## 无法由单元测试完整覆盖的场景

### 1. 真实浏览器中的跨页面配置刷新

原因：依赖真实登录会话、Next.js 客户端缓存刷新、数据库迁移和浏览器布局。

手动步骤：

1. 使用全局管理员打开 `/admin/membership`。
2. 修改 Starter 的月事件数并保存，确认版本增加且刷新后保留。
3. 打开 `/membership/upgrade`，确认 Starter 事件数立即显示新值。
4. 恢复原值并确认升级页同步恢复。

检查结果：

- [x] `500000 -> 650000` 保存为版本 1，升级页显示 `Events: 650K`。
- [x] 恢复为 `500000` 后保存为版本 2。
- [x] 初始为开启状态的布尔权益可以关闭，放弃修改后恢复。
- [x] 1280px 桌面宽度可同时比较五档套餐。
- [x] 390 x 844 视口无整页横向溢出，矩阵在面板内横向滚动。
- [x] 测试账号已恢复为普通用户。

### 2. 外部支付金额同步

原因：PayPal 订阅金额由 PayPal Plan ID 决定。后台价格是 Amami 展示和运营配置，不能安全地自动修改已存在的外部计划。

手动检查：

1. 修改后台显示价格，确认升级页价格更新。
2. 确认 PayPal 外部 Plan ID 和已存在订阅金额未被修改。
3. 若运营需要实际调价，先在支付商创建新计划，再更新对应环境变量映射。

### 3. 尚未上线能力的权益计量

原因：MCP 日调用、API 速率、AI 分析、AI 报告、告警规则、Webhook、邮件、Slack 和 SSO 等能力尚无统一的生产计量入口。

当前行为：

- 管理端以 `Published target` 标识这些权益。
- 数量会动态同步到升级页，但不会伪装成已经执行的后端拦截。
- 网站、成员、事件、Goal、数据保留和 CSV 等现有入口继续标识为 `Enforced`。

建议补充：对应能力上线时，在其服务入口读取 `getMembershipConfig()`，增加原子计量与精确边界测试，再把状态改为 `Enforced`。

### 4. 多实例缓存故障窗口

原因：Redis 不可用时，各实例使用 60 秒本地缓存保证服务可用，无法通过单进程单元测试模拟所有部署拓扑。

检查清单：

- [ ] 在预发布环境停止 Redis 后保存配置，确认当前实例立即生效。
- [ ] 确认其他实例最迟在 60 秒后读取新配置。
- [ ] Redis 恢复后确认共享缓存重新建立。

## 后续建议

- 为会员配置增加审计日志、变更原因和回滚版本。
- 支付商调价流程成熟后，将“显示价格”和“支付计划映射”放入同一运营发布检查单。
- 为多实例环境增加配置版本指标和缓存滞后告警。
