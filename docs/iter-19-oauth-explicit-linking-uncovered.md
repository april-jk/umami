# 迭代 19：OAuth 显式账户绑定与真实邮箱 - 未覆盖测试说明

## 未覆盖场景

- 真实 Google 与 GitHub provider 的授权页、回调 URI、邮箱变更和邮箱回收生命周期无法由单元测试替代。
- Redis 一次性凭据在真实集群的失效、故障转移和跨节点原子 `getDel` 需要在预生产验证。
- Prisma 唯一约束在真实 PostgreSQL 并发请求下的竞态处理未做集成演练；单元测试已覆盖 `P2002` 路径。
- `src/queries/prisma/user.ts` 是历史大型模块，定向覆盖的 OAuth 分支通过，但整个文件历史语句覆盖率为 40.84%，不应将其误读为本次新增分支未测。
- 确认页的浏览器密码管理器、无障碍与不同浏览器 fragment 保留行为需要真实浏览器测试。

## 手动测试步骤

1. 在预生产创建本地密码账号 `user@example.com`，使用同一已验证 Google 和 GitHub 邮箱首次 OAuth 登录，确认到达 `/oauth/link`，没有登录原账号或创建随机用户名账号。
2. 在确认页输入错误账号或错误密码，确认没有 OAuthAccount 绑定、没有写入登录 token；重新开始 OAuth 后，使用正确密码账号确认绑定成功。
3. 对已绑定 provider 身份再次登录，确认直接登录既有用户；尝试将同一 provider 身份绑定到另一账号，确认被拒绝。
4. 等待超过 10 分钟或重复使用同一绑定链接，确认接口返回过期/无效凭据。
5. 对 Google 和 GitHub 各执行一次并发绑定演练，确认唯一键冲突不会覆盖原绑定。

## 建议补充的测试用例

- 在预生产使用 Playwright 驱动真实 OAuth sandbox 或受控测试账号，验证 fragment 不进入服务端日志、历史或 referrer。
- 为 Redis 集群与 PostgreSQL 添加并发集成测试，验证一次性票据和 `(provider, providerAccountId)` 的唯一性。
- 实现邮箱验证与密码重置后，增加账号恢复、邮箱变更和解绑的安全回归用例。
